Class Database

Implements a Koji database for the backend of your Koji.

A Koji database is a key-value store that is included with each project on Koji. For more information, see the Koji database developer guide.

Static valueTypes

valueTypes: { increment: (number: number) => { _updateType: string; value: number } } = ...
  • Defined in backend/database/index.ts:120

Increments a numeric value by a specified amount. You can use this method to increment numeric values in the database with a single request. See update.

example

// Increment a value (using a positive integer)
const updatedDoc = await database.update('collection', 'document', {
  'myValue': KojiBackend.Database.valueTypes.increment(1),
}, true);

// Decrement a value (using a negative integer)
const updatedDoc = await database.update('collection', 'document', {
  'myValue': KojiBackend.Database.valueTypes.increment(-1),
}, true);

    example

 array1: 'newValue1',

}, true);

}

update

  • update(collection: string, documentName: string, documentBody: any, returnDoc?: boolean): Promise<any>
    • Defined in backend/database/index.ts:375

    Updates the specified data for an entry in the database collection.

    This method updates only the values that are specified in `documentBody`. If other values exist in the entry, they are not changed. If no existing entry matches the `documentName`, a new entry is created with the specified `documentName` and `documentBody`.

    example
    const myData = await database.update('myCollection', 'myDocument', {
 'myData1': 'myValue1',
 'myData2': 'myValue2'
});

    Parameters

    • collection: string

      Name of the collection.

    • documentName: string

      Name of the entry.

    • documentBody: any

      New data.

    • Optional returnDoc: boolean

      Whether to return the updated entry as a response.

    Returns Promise<any>

    An HTTP status code indicating whether the request was successful, or the updated entry if returnDoc was set to true.
